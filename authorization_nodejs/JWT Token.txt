JWT Token это некаторая строка каторая состоит из 3 часей 
заголовок header полезные данные payload и подпись signature

header jwt содержит информацию о том как должна вычесляться подпись
определяет алгоритм хеширования

payload полезные данные каторые хроняться внутри jwt
эти данные так же называют jwt claims заявки туда можно поместить id пользователя его роли username и тд.

signature
допустим есть некаторый секретный ключ о катором знает только сервер и по этому ключу в сигнатуру шифруются какие то данные 
и именно по этому ключу сервер потом может этот токен расшифровать


есть клиент и есть сервер 
клиент отправляет запрос на авторизацию с email и password
сервер генерирует токен вшивает в него id пользователя и возвращает обратно
теперь на клиенте есть токен
на примере облачного хронилеща 
пользователь хочет плучить список файлов он отправляет get запрос по указанному URL и в headers Authorization: "Bearer TOKEN" помещает токен
далее сервер получает токен распарсивает его по секретному ключу достает от туда id пользователя по этому id находит файлы в бд и возвращает
обратно клиенту
тоесть для идентификации пользователя достаточно иеть только токен

если токен украдут? 
Access Token токен доступа к этому токену еще можно прикрутить refresh Token каторый будет каждый раз перезаписывать токен доступа
